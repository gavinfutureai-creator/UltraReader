## 一、核心理念

### 1.1 LLM-First 设计原则

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   ╔═══════════════════════════════════════════════════════════════╗   │
│   ║                                                               ║   │
│   ║     🤖 LLM (主角)          ⚙️ 工程 (辅助)                     ║   │
│   ║     ──────────────────     ─────────────────                  ║   │
│   ║     • 理解书籍内容         • 读取电子书文件                    ║   │
│   ║     • 提取知识实体         • 格式化输出                        ║   │
│   ║     • 建立实体关系         • 存储和同步                        ║   │
│   ║     • 维护 Wiki            • 可视化和搜索                      ║   │
│   ║     • 自主决定结构         • 提供工具接口                      ║   │
│   ║                                                               ║   │
│   ╚═══════════════════════════════════════════════════════════════╝   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 核心价值

| 层次 | 角色 | 职责 |
|------|------|------|
| **LLM 层** | 主角 | 理解书籍、提取知识、自主决定结构、维护 Wiki |
| **工程层** | 辅助 | 输入输出、格式转换、存储同步、可视化 |

### 1.3 与传统方案的区别

| | 传统 RAG | UltraReader (LLM-First) |
|--|---------|------------------------|
| **核心** | 检索 + 匹配 | LLM 理解 + 自主提取 |
| **Schema** | 预定义固定 Schema | LLM 软约束自主决定 |
| **维护** | 代码维护 | **LLM 维护 Wiki** |
| **输出** | JSON/Database | Markdown Wiki |
| **工程** | 主导 | 辅助 |

## 五、Prompt 设计（LLM 的指导）

### 5.1 本体论理论基础

```
┌─────────────────────────────────────────────────────────────────────┐
│                     本体论 (Ontology) 核心思想                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   核心问题：                                                         │
│   ─────────────────────────────────────────────────────────────     │
│   "这个世界存在什么？它们之间如何关联？"                              │
│                                                                      │
│   本体论三要素：                                                     │
│   ─────────────────────────────────────────────────────────────     │
│   1. 实体 (Entity) - "存在什么"                                     │
│      └─ 不仅仅是"人物"，而是"任何存在的事物"                         │
│                                                                      │
│   2. 关系 (Relation) - "如何关联"                                   │
│      └─ 不仅仅是"朋友"，而是"任何可能的关联方式"                    │
│                                                                      │
│   3. 概念 (Concept) - "如何归类"                                    │
│      └─ 不仅仅是"书"，而是"理解的维度"                             │
│                                                                      │
│   关键洞见：                                                         │
│   ─────────────────────────────────────────────────────────────     │
│   • 实体不是预定义的，而是由书的内容决定的                           │
│   • 关系不是预定义的，而是由书的情境决定的                           │
│   • LLM 的任务是"发现"而非"填充"                                    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.2 核心理念

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                        │
│   不是: "按这个 Schema 输出 JSON"                                      │
│                                                                        │
│   而是: "请用你认为最好的方式理解这个世界"                            │
│                                                                        │
│   本体论 Prompt 的作用:                                                │
│   - 引导 LLM 思考"存在什么"                                           │
│   - 引导 LLM 思考"如何关联"                                           │
│   - 引导 LLM 思考"如何归类"                                           │
│   - 设定输出格式（Obsidian Wiki）                                     │
│   - 提供上下文                                                         │
│                                                                        │
│   软约束而非硬约束:                                                   │
│   - 定义理解维度，不定义具体类型                                       │
│   - 要求标注来源，不限制具体实体                                       │
│   - 要求建立关联，不限制关系名称                                       │
│                                                                        │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.3 知识提取 Prompt（本体论驱动）

```yaml
# configs/prompts/extraction.yaml

system: |
  你是一个本体论知识工程师，负责从书籍中构建知识本体。
  
  你的思维方式：
  
  1. 存在论视角
     - "这本书描绘了什么样的世界？"
     - "什么存在于此书中？"
     - 不要只关注"人物"，而是关注"任何存在的事物"
  
  2. 关系论视角
```

## 六、LLM 接口设计

### 6.1 LLM 抽象接口

```python
# llm/base.py

from abc import ABC, abstractmethod
from typing import AsyncIterator

class BaseLLM(ABC):
    """
    LLM 接口基类
    
    这是连接 UltraReader 与各种 LLM 服务的桥梁。
    LLM 负责核心的智能处理（理解、提取、生成）。
    """
    
    @property
    @abstractmethod
    def name(self) -> str:
        """LLM 名称/模型名"""
        pass
    
    @property
    def provider(self) -> str:
        """LLM 提供商: ollama, openai, anthropic"""
        return "unknown"
    
    @property
    def max_context(self) -> int:
        """最大上下文长度"""
        return 128_000
    
    @abstractmethod
    async def chat(
        self,
        messages: list[dict],
        *,
        temperature: float = 0.7,
        **kwargs
    ) -> str:
        """
        发送对话请求
        
        Args:
            messages: 对话消息列表
            temperature: 温度参数
            
        Returns:
            LLM 的原始输出
        """
        pass
    
    @abstractmethod
    async def stream(
        self,
        messages: list[dict],
        *,
        temperature: float = 0.7,
        **kwargs
    ) -> AsyncIterator[str]:
        """
        流式对话
        
        Yields:
            逐字输出的文本
        """
        pass
```

### 6.2 Ollama 实现

```python
# llm/ollama.py

import httpx
from typing import AsyncIterator

class OllamaLLM(BaseLLM):
    """
    Ollama 本地 LLM 实现
    
    利用本地算力，支持 qwen3.5:9b 等模型。
    """
    
    def __init__(
        self,
        base_url: str = "http://localhost:11434",
        model: str = "qwen3.5:9b",
        timeout: int = 300,
    ):
        self.base_url = base_url
        self.model = model
        self.timeout = timeout
    
    @property
    def provider(self) -> str:
        return "ollama"
    
    @property
    def name(self) -> str:
        return self.model
    
    @property
    def max_context(self) -> int:
        # qwen3.5:9b 支持 256K context
        return 256_000
    
    async def chat(self, messages: list[dict], **kwargs) -> str:
        """调用 Ollama /api/chat"""
        async with httpx.AsyncClient(timeout=self.timeout) as client:
            response = await client.post(
                f"{self.base_url}/api/chat",
                json={
                    "model": self.model,
                    "messages": messages,
                    "stream": False,
                    **kwargs
                }
            )
            return response.json()["message"]["content"]
    
    async def stream(self, messages: list[dict], **kwargs) -> AsyncIterator[str]:
        """流式调用 Ollama"""
        async with httpx.AsyncClient(timeout=self.timeout) as client:
            async with client.stream(
                "POST",
                f"{self.base_url}/api/chat",
                json={
                    "model": self.model,
                    "messages": messages,
                    "stream": True,
                    **kwargs
                }
            ) as response:
                async for line in response.aiter_lines():
                    if line:
                        data = json.loads(line)
                        if "message" in data:
                            yield data["message"]["content"]
```

```python
# writers/wiki.py

class WikiWriter:
    """
    Obsidian Wiki 格式写入器
    
    将 LLM 的输出转换为 Obsidian 兼容的 Markdown 格式。
    """
    
    def __init__(self, output_dir: Path):
        self.output_dir = output_dir
    
    def write(
        self,
        book_title: str,
        llm_output: str,
        metadata: dict,
    ) -> Path:
        """写入 Wiki 文件"""
        
        # 创建书籍目录
        book_dir = self.output_dir / self._sanitize(book_title)
        book_dir.mkdir(parents=True, exist_ok=True)
        
        # 写入 index.md
        index_path = book_dir / "index.md"
        content = self._format_index(llm_output, metadata)
        index_path.write_text(content, encoding="utf-8")
        
        # 生成 YAML 元数据
        self._write_metadata(book_dir, metadata)
        
        return book_dir
    
    def _format_index(self, llm_output: str, metadata: dict) -> str:
        """格式化 Obsidian Wiki 索引"""
        return f"""---
type: book
title: {metadata['title']}
author: {metadata.get('author', 'Unknown')}
processed_at: {metadata['processed_at']}
---

# {metadata['title']}

{llm_output}
"""
```
